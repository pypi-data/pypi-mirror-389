Custom Prompts

Custom prompts are reusable templates that can include variables, tool calls, resource references, and more. They allow you to create sophisticated prompt workflows that can be reused across your vMCP.

Basic Prompt Structure​

{
  "name": "greeting",
  "description": "Simple greeting prompt",
  "text": "Say hello to the user in a friendly manner",
  "variables": [],
  "environment_variables": [],
  "tool_calls": []
}

Variable Substitution​

Parameter Variables (@param)​

Use @param.variable_name to reference runtime parameters:

{
  "name": "personalized_greeting",
  "description": "Greet user by name",
  "text": "Hello @param.name, welcome to @param.location!",
  "variables": [
    {
      "name": "name",
      "description": "User's name",
      "required": true
    },
    {
      "name": "location",
      "description": "Location name",
      "required": true
    }
  ]
}

Usage:

{
  "prompt": "personalized_greeting",
  "arguments": {
    "name": "Alice",
    "location": "Wonderland"
  }
}

Result:

Hello Alice, welcome to Wonderland!

Configuration Variables (@config)​

Use @config.variable_name to reference environment variables:

First, add environment variables to your vMCP:

{
  "environment_variables": [
    {
      "name": "api_key",
      "value": "sk-..."
    },
    {
      "name": "base_url",
      "value": "https://api.example.com"
    }
  ]
}

Then reference them in prompts:

{
  "name": "api_request",
  "description": "Make API request",
  "text": "Make a request to @config.base_url using key @config.api_key",
  "environment_variables": ["api_key", "base_url"]
}

Tool Calls (@tool)​

Call other tools from within your prompts:

{
  "name": "weather_analysis",
  "description": "Analyze weather data",
  "text": "Get the current weather: @tool.get_weather(city='London') and analyze it",
  "tool_calls": [
    {
      "tool": "get_weather",
      "arguments": {
        "city": "London"
      }
    }
  ]
}

Dynamic Tool Arguments​

Use parameters in tool calls:

{
  "name": "flexible_weather",
  "description": "Get weather for any city",
  "text": "Weather for @param.city: @tool.get_weather(city=@param.city)",
  "variables": [
    {
      "name": "city",
      "description": "City name",
      "required": true
    }
  ],
  "tool_calls": [
    {
      "tool": "get_weather",
      "arguments": {
        "city": "@param.city"
      }
    }
  ]
}

Resource References (@resource)​

Reference resources from connected MCP servers:

{
  "name": "analyze_dashboard",
  "description": "Analyze dashboard data",
  "text": "Analyze the following dashboard: @resource.server_name://dashboard",
  "environment_variables": []
}

Prompt Composition (@prompt)​

Reference other prompts to build complex workflows:

{
  "name": "base_greeting",
  "description": "Base greeting",
  "text": "Hello, welcome!",
  "variables": []
},
{
  "name": "extended_greeting",
  "description": "Extended greeting",
  "text": "@prompt.base_greeting Now let me help you with your questions.",
  "variables": []
}

Complex Example​

Here's a comprehensive prompt using all features:

{
  "name": "comprehensive_analysis",
  "description": "Comprehensive data analysis",
  "text": """Analyze the following:
User: @param.user_name
City: @config.default_city
Weather: @tool.get_weather(city=@param.city)
Additional Info: @param.details
Dashboard Data: @resource.analytics://dashboard
""",
  "variables": [
    {
      "name": "user_name",
      "description": "User's name",
      "required": true
    },
    {
      "name": "city",
      "description": "City name",
      "required": false
    },
    {
      "name": "details",
      "description": "Additional details",
      "required": false
    }
  ],
  "environment_variables": ["default_city"],
  "tool_calls": [
    {
      "tool": "get_weather",
      "arguments": {
        "city": "@param.city"
      }
    }
  ]
}

Adding Prompts to vMCP​

Via API​

curl -X PUT http://localhost:8000/api/vmcps/{vmcp_id} \
  -H "Content-Type: application/json" \
  -d '{
    "custom_prompts": [
      {
        "name": "my_prompt",
        "description": "My custom prompt",
        "text": "Hello @param.name!",
        "variables": [
          {
            "name": "name",
            "description": "User name",
            "required": true
          }
        ],
        "environment_variables": [],
        "tool_calls": []
      }
    ]
  }'

Via MCP Client​

Once added, prompts are accessible via the MCP protocol:

from mcp import ClientSession
from mcp.client.streamable_http import streamablehttp_client

async with streamablehttp_client(mcp_url) as (read, write, _):
    async with ClientSession(read, write) as session:
        await session.initialize()

        # List prompts
        prompts = await session.list_prompts()

        # Get prompt with arguments
        result = await session.get_prompt(
            "my_prompt",
            arguments={"name": "Alice"}
        )

Best Practices​

1. Clear Naming​

Use descriptive names that explain what the prompt does:

✅ "analyze_sales_data"
❌ "prompt1"

2. Detailed Descriptions​

Help AI assistants understand when to use each prompt:

{
  "description": "Analyze sales data for a specific period and generate insights about top products, revenue trends, and customer segments"
}

3. Validation​

Mark required variables appropriately:

{
  "variables": [
    {
      "name": "data",
      "required": true  // Required field
    },
    {
      "name": "format",
      "required": false,  // Optional
      "default_value": "json"
    }
  ]
}

4. Documentation​

Include examples in descriptions:

{
  "description": "Format address. Example: @param.street, @param.city, @param.zip"
}

Common Patterns​

Data Transformation​

{
  "name": "format_user_data",
  "text": "Format: Name: @param.name, Email: @param.email, Role: @param.role"
}

Multi-Step Workflow​

{
  "name": "research_workflow",
  "text": "@tool.search(query=@param.topic) → @tool.summarize(text=@tool.search) → @prompt.format_report"
}

Conditional Logic​

{
  "text": "If @param.mode is 'detailed', provide comprehensive analysis. Otherwise, give brief summary."
}
