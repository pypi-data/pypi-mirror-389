Custom Tools Overview

vMCP allows you to create custom tools in three different types, each suited for different use cases. Custom tools extend your vMCP's capabilities beyond what connected MCP servers provide.

Tool Types​

1. Prompt Tools​

Simple, text-based tools that execute prompts with variable substitution. Perfect for formatting, templating, and simple data transformations.

Best for:

  • Text formatting and templates
  • Simple data transformations
  • Combining multiple tool calls
  • Creating reusable prompt workflows

Learn more about Prompt Tools →

2. Python Tools​

Execute Python code with full access to Python's standard library. Ideal for complex data processing, calculations, and logic.

Best for:

  • Mathematical calculations
  • Data processing and transformations
  • Complex business logic
  • Working with JSON, lists, and dictionaries

Learn more about Python Tools →

3. HTTP Tools​

Make HTTP requests to external APIs with support for various authentication methods. Perfect for integrating third-party services.

Best for:

  • REST API integration
  • External service calls
  • Fetching data from web services
  • Webhook integrations

Learn more about HTTP Tools →

Common Tool Properties​

All custom tools share these core properties:

{
  "name": "tool_name",           // Unique identifier
  "description": "What it does",  // Shown to AI
  "tool_type": "prompt|python|http",
  "variables": [],                // Input parameters
  "environment_variables": [],    // Config references
  "tool_calls": [],              // Other tools to call
  "atomic_blocks": []            // Advanced: atomic operations
}

Variable Types​

All tools support typed variables:

TypeDescriptionExample
strString/text"Hello World"
intInteger42
floatDecimal number3.14
boolBooleantrue or false
listArray of values[1, 2, 3]
dictKey-value object{"key": "value"}

Variable Definition​

{
  "variables": [
    {
      "name": "user_name",
      "description": "The user's name",
      "type": "str",
      "required": true
    },
    {
      "name": "age",
      "description": "User's age",
      "type": "int",
      "required": false,
      "default_value": 0
    }
  ]
}

Environment Variables​

Environment variables store sensitive configuration and can be reused across tools:

{
  "environment_variables": ["api_key", "base_url"]
}

Reference them in tools using @config.variable_name:

Make request to @config.base_url using @config.api_key

Tool Composition​

Tools can call other tools using the tool_calls property:

{
  "tool_calls": [
    {
      "tool": "get_weather",
      "arguments": {
        "city": "@param.city"
      }
    }
  ]
}

In the tool text/code, reference the result:

Weather data: @tool.get_weather(city=@param.city)
