Prompt Tools

Prompt tools are the simplest type of custom tool. They execute text-based prompts with variable substitution, perfect for formatting, templates, and simple workflows.

Basic Structure​

{
  "name": "simple_analyzer",
  "description": "Analyze provided data",
  "tool_type": "prompt",
  "text": "Analyze the provided data and give insights",
  "variables": [],
  "environment_variables": [],
  "tool_calls": []
}

With Variables​

{
  "name": "personalized_report",
  "description": "Generate personalized report",
  "text": "Generate a @param.report_type report for @param.user_name",
  "tool_type": "prompt",
  "variables": [
    {
      "name": "report_type",
      "description": "Type of report",
      "required": true
    },
    {
      "name": "user_name",
      "description": "User's name",
      "required": true
    }
  ]
}

With Config Variables​

{
  "name": "api_info",
  "description": "Show API information",
  "text": "API: @config.base_endpoint version @config.api_version",
  "tool_type": "prompt",
  "variables": [],
  "environment_variables": ["base_endpoint", "api_version"]
}

With Tool Calls​

{
  "name": "weather_reporter",
  "description": "Report weather",
  "text": "Weather report for Paris: @tool.get_weather(city='Paris')",
  "tool_type": "prompt",
  "tool_calls": [
    {
      "tool": "get_weather",
      "arguments": {
        "city": "Paris"
      }
    }
  ]
}

Complete Example​

{
  "name": "comprehensive_reporter",
  "description": "Comprehensive reporting tool",
  "text": "Generate Report:\nUser: @param.user_name\nDefault City: @config.default_city\nWeather Data: @tool.get_weather(city=@param.city)\nSummary: @param.summary",
  "tool_type": "prompt",
  "variables": [
    {
      "name": "user_name",
      "description": "User's name",
      "required": true
    },
    {
      "name": "city",
      "description": "City for weather",
      "required": false
    },
    {
      "name": "summary",
      "description": "Summary text",
      "required": false
    }
  ],
  "environment_variables": ["default_city"],
  "tool_calls": [
    {
      "tool": "get_weather",
      "arguments": {
        "city": "@param.city"
      }
    }
  ]
}

Use Cases​

1. Text Formatting​

{
  "text": "Format: Name: @param.name, Email: @param.email, Role: @param.role"
}

2. Template Generation​

{
  "text": "Dear @param.name,\n\nThank you for @param.action.\n\nBest regards,\n@config.company_name"
}

3. Data Aggregation​

{
  "text": "User @param.user: @tool.get_profile(id=@param.user) + @tool.get_activity(id=@param.user)"
}

Best Practices​

  1. Clear Text: Write clear, specific instructions
  2. Variable Names: Use descriptive variable names
  3. Required vs Optional: Mark variables appropriately
  4. Tool Integration: Combine with other tools for powerful workflows
